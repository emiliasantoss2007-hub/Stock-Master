# Modelo Físico

## Descrição

O Modelo Físico do banco de dados do sistema Stock Master será desenvolvido a partir do Modelo Lógico aprovado.

Esta etapa será responsável por definir a implementação do banco de dados em um Sistema Gerenciador de Banco de Dados (SGBD), incluindo:

* Estrutura das tabelas;
* Tipos de dados específicos;
* Chaves primárias;
* Chaves estrangeiras;
* Restrições de integridade;
* Índices;
* Scripts SQL de criação do banco.

## Objetivo

Transformar a modelagem lógica em uma estrutura pronta para implementação e utilização pela aplicação.

## Status

Em desenvolvimento.

## Artefatos previstos

* Script de criação do banco de dados (DDL);
* Diagrama físico;
* Documentação das tabelas e relacionamentos;
* Scripts de restrições e índices.

## Dependências

O modelo físico será elaborado com base nos seguintes artefatos:

* Modelo Conceitual;
* Modelo Lógico;
* Requisitos Funcionais;
* Regras de Negócio.
